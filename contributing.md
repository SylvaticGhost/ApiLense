# Brunches

## Master brunch

To master brunch is merged with fetures by PR from developing version brunches(dev-v.X.X.X) and deployment action(release) will perform. Normaly process is executed at the end of developing iteration.

## Developing brunches

For each version creates new dev brunch from master with version indexing and name looks like dev-v.X.X.X. In developing cycle new feature are merged to respective version brunch by PR, merging require successful testing and code review.
